- BEFORE INSERT, BEFORE UPDATE, and BEFORE DELETE triggers
- Primary keys and foreign keys

## What I understood
- I understood how to create tables based on a database diagram and how triggers can automate actions within a table. I learned how to use :NEW and :OLD to access row values inside triggers depending on the operation being performed. I also understood how to automatically store audit information such as the current date and user during insert operations. I learned how to restrict updates so that only the user who created a record can modify it and how to enforce security rules like allowing only a manager to delete records. I also understood that errors raised by triggers are intentional and help control database behavior.
